1. 저장소 관리

  1. Clone
      2. git clone ${URL}
  1. Initialize
      2. git init
git remote add origin ${URL}
 
 

2. 파일 관리

  1. add
      2. git add .
  1. commit
      2. git commit -m ${MESSAGE}
  1. push
      2. # 원격 연결 후 최초 1번만 실행
git push --set-upstream origin ${BRANCH_NAME}
# 일반적 push
git push
 
 

3. 브랜치 관리

  1. 목록 확인
      2. git branch
  1. 생성
      2. git branch ${BRANCH_NAME}
  1. 이동
      2. git switch ${BRANCH_NAME}
git checkout ${BRANCH_NAME} # 구버전
  1. 삭제
      2. git branch -D ${BRANCH_NAME} # 로컬 브랜치 삭제
git push origin --delete ${BRANCH_NAME} # 원격 브랜치 삭제
  1. 상태 확인
      2. git status
 
 

4. pre-commit

  1. 설치
      2. pip install pre-commit
  1. .pre-commit-config.yaml 생성
      2. repos:
- repo: https://github.com/pre-commit/pre-commit-hooks
  rev: v3.3.0
  hooks:
    - id: no-commit-to-branch
      args: ['--branch', 'main']
  1. 저장소 내 설치
      2. pre-commit install
  1. main brach commit 실패 확인
 
 

5. 로컬 폴더 원격 저장소 설정

  • 아래의 명령어 순서대로 실행한다
      • git init
git remote add origin ${GIT_URL}

git add .
git commit -m "${commit_message}"
git push --set-upstream origin master
 
 

6. 원격 브랜치 commit 메시지 수정

  1. 원격 브랜치 commit 메시지 수정
  • 아래의 명령어 순서대로 실행한다
      • git commit --amend -m "${commit_message}"
git push --force
 
 

7. 원격 브랜치 commit 기록 삭제

  • git log로 기록 살펴보기
  • 다음의 명령어로 리셋하기
      • git reset HEAD^~${COUNT}
git reset HEAD^~1
  • push로 원격 저장소 변경 업데이트하기
      • git push -f
  • 로그 삭제 확인하기
